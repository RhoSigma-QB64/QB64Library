Base64Encode$ (function)

Encode the given string message data into a Base64 string. The data can be any sequence of bytes inclusive zeros and control chars. There are also options to determine how the encoding shall be done.

SYNTAX:

encoded$ = Base64Encode$ (message$, wrapLen%, padFlag%, lineMode%)

INPUTS:

message$ (STRING)

  • The message data you want to encode into Base64.
  • Each <LF> code (CHR$(10)) does end a line, if encoding in line mode.

wrapLen% (INTEGER)

  • This value specifies the number of chars in the encoded Base64 string, after which a line break shall be inserted. If this is less or equal to zero (0), then no line breaks are inserted at all.
  • Note that the line breaks are Unix style, hence <LF> only (CHR$(10)).
  • This option is not valid in line mode, any given value is ignored then.

padFlag% (INTEGER)

  • This flag controls whether the one or two pad characters are included at the end of encoding. If zero (0), then no padding is done, any other value will enable padding.

lineMode% (INTEGER)

  • If line mode is enabled (any value <> 0), then each line found in the input message string is separately encoded into a single Base64 string. The <LF> codes are preserved, hence the encoded result will have the same line structure as the input message.
  • Each of the resulting Base64 encoded lines could be decoded separately, or multiple/all lines could be decoded using the line mode again.

RESULT:

encoded$ (STRING)

  • The encoded Base64 string, according to the given options.
