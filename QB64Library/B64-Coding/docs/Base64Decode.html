Base64Decode$ (function)

This will decode a Base64 string back into the original data. There are choices about what to do with illegal characters or malformed strings. This function has a strict flag to indicate that the input must be a single valid Base64 string with no illegal characters. Otherwise there is best effort to decode around illegal characters which are preserved in the output. So "TWFyeQ==.aGFk.YQ.bGl0dGxl.bGFtYg==" decodes to "Mary.had.a.little.lamb" with five separate Base64 strings decoded, each separated by the illegal character (dot). In strict mode the first dot would trigger a fatal error. The four whitespace characters <CR>, <LF>, <TAB> and <SPACE> are silently ignored in any case.

SYNTAX:

message$ = Base64Decode$ (encoded$, strict%, lineMode%)

INPUTS:

encoded$ (STRING)

  • The Base64 encoded string data, which shall be decoded again.
  • Each <LF> code (CHR$(10)) does end a line, if decoding in line mode.

strict% (INTEGER)

  • If this flag is set (any value <> 0), then the input must be a single valid Base64 string with no illegal characters, no extra padding, and no short segments. Otherwise an error message is returned.

lineMode% (INTEGER)

  • If line mode is enabled (any value <> 0), then each line found in the input data must represent a single valid Base64 string and is separately decoded back into the respective original data line. The <LF> codes are preserved, hence the decoded result will have the same line structure as the Base64 input data.

RESULT:

message$ (STRING)

  • The decoded original message string, according to the given options.
  • In strict mode it may be one of the error messages listed below.

ERRORS:

B64DEC_ERROR_DAP
  • Data after pad character.
B64DEC_ERROR_PCW
  • Pad character in wrong place.
B64DEC_ERROR_SSB
  • Single symbol block not valid.
B64DEC_ERROR_BCI
  • Bad character in input string.
